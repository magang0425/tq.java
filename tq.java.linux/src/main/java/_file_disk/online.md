## 线上问题解决

### 磁盘占用告警
1. df -hl
    - 查看各个分区的磁盘占用情况
2. 发现 /dev/vda1 还有 7% 剩余
3. du -sh /usr/* | sort -rn 获取文件大小排序
    - 没有发现大文件 > 10G
3. 搜索 /dev/vda1 看不到
    - 发现是因为 有进程一直占用文件
4. 用lsof检查后才发现原因是,有文件被删除,而进程还活着,因而造成还占用空间的现象
    - lsof |grep delete
5. 根据lsof列出的进程号,kill这些进程后,空间就释放出来了
