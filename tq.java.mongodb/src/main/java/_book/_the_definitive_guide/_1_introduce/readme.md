## 介绍

### 为什么
1. 易于使用
    - 不再有预定义模式 
    - 文档由键值对组成
    - 不再有固定的类型和大小
2. 易于扩展
    - 横向扩展
    - 面向文档的数据类型使它很容易的在多台服务器之间进行数据分割
    - 自动处理跨集群的数据和负载
    - 自动重新分配文档
3. 丰富的功能
    - 索引
        - 通用二级索引
        - 唯一索引
        - 复合索引
        - 地理空间索引
        - 全文索引
    - 聚合
        - 聚合管道
        - 通过简单的片段创建复杂的聚合
    - 特殊的集合类型
    - 文件存储
4. 卓越的性能
    - 对文档进行动态填充
    - 预分配数据文件以利用额外的空间来换取稳定的性能
    - 尽可能多的内存用作缓存, 为每次查询自动选择正确的索引
    - **试图保留关系型数据库的许多特性, 不追求具备关系型数据库的所有功能**
